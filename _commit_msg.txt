fix: powershell import error + hide git updater for packaged installs

Two issues fixed:

1. Agent init crash: 'No module named tools.environments.powershell'
   - terminal_tool.py imported PowerShellEnvironment at the top level,
     but tools/environments/powershell.py doesn't exist in the repo
   - Made the import lazy (try/except ImportError) so the agent can
     init without it; raises a clear RuntimeError only if someone
     actually configures TERMINAL_ENV=powershell

2. Dual auto-updater confusion for packaged (NSIS) installs
   - The app had two updaters: electron-updater (GitHub Releases,
     for packaged installs) and git-based checkUpdates/applyUpdates
     (git fetch + hermes update, for source installs)
   - Both showed up simultaneously for packaged installs, confusing
     users with two different update mechanisms
   - Added isPackaged to the version IPC response
   - For packaged installs: hide git-based update UI (status bar
     behind-indicator, about-settings update section, command-center
     update button, update toast notifications)
   - electron-updater remains the sole update mechanism for packaged
     installs; git-based updater still works for source/dev installs
