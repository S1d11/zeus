v0.21.2: Fix wake word + boot straight into voice mode

Wake word fix:
- Created missing scripts/wake_word.py (the file the wake-word module
  spawns but which never existed in the repo — this was the root cause
  of wake word not working at all)
- Uses speech_recognition + PyAudio to continuously listen for "zeus"
  or "hey zeus", printing JSON events to stdout
- Calibrates for ambient noise on startup, dynamically adjusts energy
  threshold, and recovers from transient errors

Wake word → voice mode:
- When the wake word is detected, the main process now sends an IPC
  event (zeus:wake-word:detected) to the renderer instead of just
  showing/focusing the window
- New requestVoiceStart() event in composer/focus.ts (idempotent —
  only starts voice mode, doesn't toggle it off if already active)
- Desktop controller listens for the wake word event and dispatches
  requestVoiceStart() to the composer
- New onComposerVoiceStartRequest handler in composer starts voice
  conversation without toggling off
- Saying "Hey Zeus" now boots straight into voice mode

Auto-start on boot:
- Wake word listener now auto-starts on app boot if the user has it
  enabled in settings (previously only started when toggled manually)
- New autoStartWakeWord() in general-settings store checks status
  before starting to avoid double-start

IPC additions:
- preload.cjs: onWakeWordDetected callback registration
- global.d.ts: onWakeWordDetected type definition
- main.cjs: mainWindow.webContents.send('zeus:wake-word:detected')
